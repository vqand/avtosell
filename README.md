# 5VITO — Маркетплейс по продаже автомобилей

Полнофункциональное веб-приложение по продаже автомобилей (в стиле Avito),
разработанное по техническому заданию из [`prompt.md`](prompt.md) и
предоставленным макетам Figma «5VITO».

| Слой       | Технология                                  |
|------------|---------------------------------------------|
| Фронтенд   | React 18 + Vite, React Router               |
| Бэкенд     | PHP 8.1+ REST API (собственный MVC, PDO)    |
| База данных| MySQL 8 / MariaDB 10.4+                      |
| Авторизация| JWT (HS256), хеширование паролей bcrypt     |

> **Объём реализации:** это работающий **MVP-каркас** — полноценная рабочая
> основа (авторизация, каталог с поиском и фильтрами, CRUD объявлений, загрузка
> изображений, избранное, личный кабинет, полная админ-панель). План
> дальнейшего развития см. в [`docs/architecture.md`](docs/architecture.md).

│   │   ├── Controllers/       # Auth, Car, Favorite, Brand, Upload, Admin
│   │   └── Models/            # модели данных (PDO)

## Требования к окружению

- **PHP 8.1+** с включённым `pdo_mysql` (входит в комплект XAMPP)
- **MySQL / MariaDB** (XAMPP)
- **Node.js 18+** и npm

> Пользователям XAMPP: PHP CLI находится в `C:\xampp\php\php.exe`, а MySQL — в
> `C:\xampp\mysql\bin\mysql.exe`. Либо добавьте их в `PATH`, либо используйте
> полные пути в командах ниже.

## 1. Настройка базы данных

Запустите MySQL (XAMPP Control Panel → MySQL → Start), затем выполните:

```bash
mysql -u root -p < database/schema.sql
mysql -u root -p fvito < database/seed.sql
```

Команды создают базу данных `fvito` и загружают демо-данные.

**Демо-аккаунты** (пароли уже захешированы в `seed.sql`):

| Роль          | Логин              | Пароль        |
|---------------|--------------------|---------------|
| Администратор | `admin@5vito.ru`   | `admin123`    |
| Пользователь  | `ivan@example.com` | `password123` |
| Пользователь  | `maria@example.com`| `password123` |

## 2. Бэкенд (PHP API)

```bash
cd backend
cp .env.example .env        # при необходимости отредактируйте DB_USER / DB_PASS
php -S localhost:8080 -t public public/router.php
```

API будет доступен по адресу `http://localhost:8080/api`. Проверка:

```bash
curl http://localhost:8080/api/health      # → {"status":"ok"}
```

> Composer **не обязателен** — бэкенд использует встроенный автозагрузчик и
> реализацию JWT без внешних зависимостей. (`composer install` тоже работает,
> если нужен оптимизированный PSR-4 автозагрузчик; `composer serve` — это
> ярлык для команды выше.)

Для продакшена настройте Apache/Nginx с корнем веб-сервера в `backend/public/`
(файл `.htaccess` перенаправляет все запросы на `index.php`).

---

## 3. Фронтенд (React)

```bash
cd frontend
npm install
npm run dev                  # → http://localhost:5173
```

Dev-сервер Vite проксирует `/api` и `/uploads` на `http://localhost:8080`,
поэтому в разработке не нужно настраивать CORS. Откройте
**http://localhost:5173** и войдите под одним из демо-аккаунтов.

Для продакшена: `npm run build` (результат в `frontend/dist/`), затем
раздавайте статические файлы и укажите `VITE_API_BASE` равным адресу API.

---

## Запуск в Windows одной командой

Если PHP установлен через winget, а MySQL — как служба `MYSQL80`, можно поднять
весь стек скриптом:

```powershell
powershell -ExecutionPolicy Bypass -File start-dev.ps1
```

Скрипт запускает службу MySQL, PHP API и Vite и выводит адреса.

## Возможности

**Публичные / пользовательские**
- Регистрация, вход (по телефону *или* почте), сессия на JWT, редактирование профиля
- Каталог автомобилей с полнотекстовым поиском и фильтрами (бренд, цена,
  страна бренда, привод, тип топлива, КПП, год, пробег) + сортировка
- Карточка автомобиля с полной спецификацией, галереей фото, контактами продавца
- Добавление / удаление из избранного
- Создание / редактирование / удаление своих объявлений с загрузкой фото
- Жалоба на объявление; оформление заявки на покупку

**Админ-панель** (`/admin`, только роль администратора)
- Дашборд со статистикой и недавней активностью
- Управление пользователями (просмотр, смена роли, блокировка, удаление)
- Модерация объявлений (одобрить / отклонить / удалить, фильтр по статусу)
- Просмотр жалоб
- Управление брендами и моделями
- Журнал действий администраторов

## О безопасности

- Пароли хешируются `password_hash()` (bcrypt).
- Все SQL-запросы — через **подготовленные выражения** (PDO, эмуляция выключена) —
  защита от SQL-инъекций.
- JWT проверяется с использованием `hash_equals` (защита от тайминг-атак); срок
  действия токена контролируется.
- Ролевой контроль доступа (RBAC) через middleware на каждом маршруте админки.
- Валидация загрузок (определение MIME, ограничение размера, случайные имена файлов).
- React по умолчанию экранирует вывод (защита от XSS); API отдаёт только JSON.

Важно: перед реальным развёртыванием обязательно задайте надёжный `JWT_SECRET`,
отдельного пользователя БД (не `root`), `APP_DEBUG=false` и работайте по HTTPS.
